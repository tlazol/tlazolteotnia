Vercel から Cloudflare に移行しました。その備忘録です。

ちなみに人力で書いています。最近AIに変わりに文章を書かせすぎて、自分の中の言語化能力が衰えてきたのを感じたので、リハビリです。

自分の知的貯金削ってAI使っていたら、ざまぁないですね。

# なぜ移行したか

昨今の動きを見るに、もうそろそろ cloudflare 使ってないとやばいなと思うところがあったからというのが一番ですが、単純にもう Next.js じゃなくて ReactRouter で作っているし、Vercel じゃなくていいなと思えたのもあります。

あとランニングコストを抑えたかったというのもあります。これまでの Vercel + squarespace から Cloudflare 一本にしました。月額2000円ほど浮いたので嬉しいです。

# 移行方法

AIコーディングで「Vercel から Cloudflare に移行したい」と言って plan を軽く練るだけでいけました。

このサイトはマークダウンファイルを読み取ってそのまま記事ページにするロジックなのですが、Vercel で使えたファイル読み込み方法が Cloudflare で使えなかったりしました。

しかし、AI君が問題なく対応してくれました。さすがやでぇ。codex君と claude君の合わせ技です。あと全然関係ないけど、claude codeとcloudeflareでクラウドのスペルが違うのまじでタイポする。

具体的に書けば

- Vercelプリセット、@vercel/react-router、@vercel/og、Docker関連ファイルを削除。
- wrangler.jsonc と workers/app.ts を追加。
- Cloudflare Viteプラグインを導入し、0rga.org のカスタムドメインへデプロイする構成に変更。
- SSRをNodeのストリーム／暗号APIから、renderToReadableStream とWeb Crypto APIへ移行。
- React Routerを7.15から8.1へ更新。Node.js 22.22以上、TypeScript 6、Vite 8.1系を前提とする構成
  へ更新。
- ブログ記事の読み込みをNodeのファイルシステムから import.meta.glob に変更し、MarkdownをWorker
  バンドルへ含められるようにした。

といった感じです。

# og 画像の生成方法の変更

以前は @vercel/og のモジュールを使用して動的に記事ページのogイメージを作っていましたが、「別に動的じゃなくていいのでは？」と思い、og 画像生成スクリプトを作り、git push 時に実行されるようにしました。

動的だと、Cloudflare workers の cpu 使用時間も増えるとのことでしたので、これぐらいの規模の個人サイトなら静的で十分だなといった所です。

# ついでのデザイン変更

元 Web デザイナーとして、やはりサイトのデザインはころころと変えたい所存。

という事で今回はタイムラインっぽくしてみました。なじみのあるデザインになったので、UI的に迷わないというメリットもあるし、「オマージュしてやったぞ」という嬉しみもあります。

ネオンカラーのデザインは単純に自分の好みです。いいよね、ネオンカラー。
